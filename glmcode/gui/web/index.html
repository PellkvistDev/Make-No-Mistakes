Appearance

      

        

          Cinematic backdrop behind the glass

          that repaint constantly — the biggest cause of general sluggishness. Turn OFF for the

Browser

      
Settings for the agent's control_chrome browser.

      

        Run control_chrome headless — the browser works
          invisibly. Watch it live in the Browser panel instead. Off by default so you can
          see (and take over) the real window.

        

      

        Keep the agent browser's cookies & logins between sessions in a
          dedicated profile (never your personal browser) — pause it, sign in to the sites you
          want it to use once, and it stays signed in. Off = a fresh throwaway profile every time.

        

      

        Driving web pages is the hardest job for the small free model. Route
          just the browser agent to a stronger configured model (e.g. a bigger local Ollama
          model) for much more reliable browsing.

        

      

        Wipe the remembered profile — logs the agent browser out of everything.

        

    


    

      
Permissions

      

        
        
        
      

      
Ask: approve every edit & command. Auto-edit: file edits and
         read-only commands (git status, ls, cat, grep…) run automatically; other
         commands still ask. Full auto: everything runs without asking.

      

        Access rules — per-path exceptions to the mode…
        
Override the mode for specific files or folders, whatever it's set to.
           Allow auto-approves edits there (even in Ask); Ask always
           prompts; Deny blocks edits entirely (even in Full auto). Most protective rule
           wins. Globs: src/** a whole folder, *.py a segment,
           .env that filename anywhere.

        

        

          
          

            
            
            
          

          

            
          

        

      

    


    

      
Vision

      

        
        
      

      
Describe: the free GLM vision model narrates images for the

         straight to the chat model — use this when your model is multimodal; the
         free default falls back to GLM vision automatically.

APIs

      
The API this chat talks to — click one to select it. New chats
         always start on the free z.ai default. Any OpenAI-compatible endpoint works
         (OpenRouter, Ollama, LM Studio, paid APIs).

      

      

        
        
      

      

        
        
        
        
        

          
          
        

      

      

        MCP servers — external tools for the agent…
        
Connect Model Context Protocol servers (stdio). Their tools
           appear to the agent alongside the built-in ones, permission-gated like
           any other tool. Example command:
           npx -y @modelcontextprotocol/server-filesystem C:\projects

        

        

          
          
          

            
          

        

      

      

        Slash commands — reusable prompts…
        
Save a prompt you use often, then run it in the composer with
           /name. Use $INPUT in the template to drop in text you
           type after the command (otherwise it's appended).

        

        

          
          
          

            
          

        

      

      

        Also on the chip in the title bar. Higher = better answers, slower.

        

          
          
          
          
        

      

      
Low: no reasoning, answers instantly.
        Medium: the model thinks before answering (default).
        High: after answering, an independent reviewer checks the actual diff
        (blind to the reasoning that produced it) and the model fixes what it flags.
        Max: reviews repeatedly (up to 3×) until it's approved.

      

        Display the thinking stream in chat

        

      

        When the agent edits files but runs nothing to check them, ask it once to
          verify (run tests / the affected code). Turn off if you'd rather it just edits without
          running commands.

        

      

        Make search_code match by meaning, not just keywords, using a local
          embedding model. Turning it on downloads a small model (~90MB) once; until it's ready,
          keyword search is used. Fully offline after setup.

        

      

        After a turn that edits files, run the project's tests; if they fail, keep
          fixing until they pass (up to a few tries) or report where it's stuck. Only runs when there
          are tests to run — never loops on small tasks. Off by default.
